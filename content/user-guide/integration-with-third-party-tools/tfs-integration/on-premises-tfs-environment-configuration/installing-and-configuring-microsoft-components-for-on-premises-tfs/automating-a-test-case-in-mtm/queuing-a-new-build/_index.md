After creating a build definition, the new build must be queued.

After creating and configuring a build definition \([learn more](/user-guide/integration-with-third-party-tools/tfs-integration/on-premises-tfs-environment-configuration/installing-and-configuring-microsoft-components-for-on-premises-tfs/automating-a-test-case-in-mtm/creating-and-configuring-a-build-definition/)\), run it to create a build instance to be associated with a test plan:

-   **[Queueing a new XAML build](/user-guide/integration-with-third-party-tools/tfs-integration/on-premises-tfs-environment-configuration/installing-and-configuring-microsoft-components-for-on-premises-tfs/automating-a-test-case-in-mtm/queuing-a-new-build/queueing-a-new-xaml-build)**  
After creating an XML build definition, the new build must be queued.
-   **[Queueing a new TFBuild](/user-guide/integration-with-third-party-tools/tfs-integration/on-premises-tfs-environment-configuration/installing-and-configuring-microsoft-components-for-on-premises-tfs/automating-a-test-case-in-mtm/queuing-a-new-build/queueing-a-new-tfbuild)**  
Once you've defined your build processes by creating a TFBuild definition, the new build definition must be queued.
